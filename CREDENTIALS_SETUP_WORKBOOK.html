☁️ Google Drive OAuth Setup

Step-by-Step Guide:
  1. Go to https://console.cloud.google.com/
  2. Click "Select a Project" → "New Project"
  3. Name: Semptify → Create
  4. Search for "Google Drive API" → Click → Enable
  5. Go to "Credentials" (left sidebar)
  6. Click "Create Credentials" → "OAuth client ID"
  7. Choose "Web application"
  8. Add Authorized JavaScript origins: http://localhost:8000
  9. Add Authorized redirect URIs (see below)
  10. Click "Create" → Copy Client ID and Client Secret

Redirect URIs:

http://localhost:8000/storage/oauth/google/callback
http://yourdomain.com/storage/oauth/google/callback
Credentials obtained

📦 Dropbox OAuth Setup

Step-by-Step Guide:
  1. Go to https://www.dropbox.com/developers/apps
  2. Click "Create app"
  3. Choose:
    • API: Scoped access
    • Type: Full Dropbox
  4. App name: Semptify
  5. Go to "Settings" tab → OAuth 2
  6. Add Redirect URIs (see below)
  7. Go to "Permissions" → Enable files.content.read, files.content.write, files.metadata.read
  8. Copy App Key and App Secret from "Settings"

Redirect URIs:

http://localhost:8000/storage/oauth/dropbox/callback
http://yourdomain.com/storage/oauth/dropbox/callback
Credentials obtained

☁️ OneDrive / Microsoft OAuth Setup

Step-by-Step Guide:
  1. Go to https://portal.azure.com/
  2. Search for "App registrations" → Click
  3. Click "New registration"
  4. Name: Semptify
  5. Supported account types: Accounts in any organizational directory
  6. Redirect URI: Select "Web" and add URI (see below)
  7. Register → Copy "Application (client) ID"
  8. Go to "Certificates & secrets" → New client secret
  9. Copy the secret value (not ID)
  10. Go to "API permissions" → Add "Microsoft Graph" → "Files.ReadWrite.All"

Redirect URIs:

http://localhost:8000/storage/oauth/onedrive/callback
http://yourdomain.com/storage/oauth/onedrive/callback
Credentials obtained

🔒 Cloudflare R2 Setup (Production System Storage)

⚠️ Important: R2 is required for production deployments and stateless server instances. Local development can use SQLite with in-memory storage.
Step-by-Step Guide:
  1. Go to https://dash.cloudflare.com/
  2. Log in → Select account
  3. Left sidebar → "R2" → "Create bucket"
  4. Bucket name: semptify-server-prod
  5. Leave location as "Default"
  6. Create → Save bucket name and region
  7. Go to "Settings" → Copy endpoint URL
  8. Left sidebar → "API Tokens" → Create API token
  9. Select "R2" scope → "All buckets" permissions → Generate
  10. Copy Account ID, Access Key ID, Secret Access Key

🗄️ Database Configuration

Development: SQLite (local, automatic)
Production: PostgreSQL (recommended)

For Development (SQLite) - Automatic

SQLite database automatically created at: ./semptify.db

No configuration needed for local development.

For Production (PostgreSQL)

Option A: Managed Service (Recommended)
  • Use AWS RDS, Azure Database, or Heroku Postgres
  • Create database named semptify
  • Create user and set password
  • Note the connection details
Option B: Self-Hosted
  • Install PostgreSQL 13+
  • Create database: createdb semptify
  • Create user and password
# SQLite Example (Development) DATABASE_URL=sqlite+aiosqlite:///./semptify.db # PostgreSQL Example (Production) DATABASE_URL=postgresql+asyncpg://semptify_user:secure_password@db.example.com:5432/semptify

✅ Final Deployment Checklist

Before Going Live:

  • All OAuth credentials configured (Google, Dropbox, OneDrive)
  • R2 bucket created and credentials tested
  • At least one AI provider configured (Anthropic recommended)
  • PostgreSQL database connection tested
  • Secret key generated and stored securely
  • Admin PIN set (change from default)
  • SECURITY_MODE set to "enforced"
  • .env file created with all credentials
  • .env file added to .gitignore (DO NOT COMMIT)
  • Server started and tested: http://localhost:8000/onboarding
  • SSL/TLS certificate configured (production only)
  • Firewall rules configured
  • Monitoring/logging set up
  • Backups scheduled

⚠️ Security Reminders

  • Never commit .env file to version control
  • Use strong, unique passwords for all services
  • Rotate API keys regularly
  • Use environment variables in production, not hardcoded values
  • Enable 2FA on all external service accounts
  • Store credentials in a secure vault (1Password, LastPass, etc.)
  • Back up SECRET_KEY and ADMIN_PIN in a secure location
  • Monitor API usage and set billing alerts
